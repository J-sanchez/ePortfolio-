Description

For my ePortfolio, I selected the Inventory Tracking System from DAD 220: Intro to Structural Database Environments. The original project was a simple MySQL database managed via terminal, where I created basic tables and relationships. While functional, it lacked interactive features for real-world use.

In the enhanced version, I transformed the ERD into a fully functional Oracle APEX application. Key features include:

  • Database Implementation: Converted ERD to SQL tables with necessary relationships.
  • CRUD Operations: Allowed users to create, read, update, and delete inventory, orders, and customer data.
  • Reporting Dashboard: Created SQL queries for reports, tracking low-stock items and sales trends.
  • User Interface: Developed user-friendly forms for easy data input and management.
  • Data Seeding: Seeded the database with sample data to demonstrate functionality.

Key Skills Demonstrated:

  • Database Design: Designed and implemented a relational database, ensuring data integrity.
  • SQL Querying: Wrote queries for reporting and analysis, solving real-world problems.
  • Application Development: Built an interactive Oracle APEX system for data management.
  • User Interface Development: Designed an intuitive and functional user interface.
  • Security Considerations: Followed best practices to handle data securely.

This project allowed me to apply database management, SQL, and application development skills, turning a conceptual ERD into a practical inventory system. In the future, I'd enhance the system with more advanced features and improve security.
